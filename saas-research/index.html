撞球店雲端 SaaS 系統研究

研究全球撞球店家所用的管理系統——中國、東南亞與歐美的代表廠商、功能差異與市場趨勢，並推導出我們若要打造一套雲端 SaaS 服務應具備的功能、競品定位與可施工的技術架構。

🇨🇳 中國 · 無人自助 🌏 東南亞 · 防作弊藍海 🌎 歐美 · POS＋預約 📡 IoT 燈控盒設計 📑 28＋ 一手來源

1結論摘要

先看這五點，全篇精華濃縮於此。

防「時間財」流失是亞洲第一痛點。 店員偷算檯費、少報時間、私吞飲料錢。越南文章直指「約 80% 撞球店每天都在被員工偷錢」。精確計時＋物聯網燈控（開檯才亮燈）＋防作弊稽核是入場門票。
中國已走到無人值守自助共享球房。 靠 90%+ 行動支付與無人經濟紅利，掃碼開檯、遠端管理、自動分潤成主流，並疊加 AI 裁判／自動計分／精彩剪輯／人臉門禁。
東南亞仍以「有人服務＋POS＋防作弊」為主。 雲端與自助化落後中國一個世代，是明顯藍海。
在地化是生死線。 馬來西亞要 DuitNow QR＋LHDN 電子發票、越南要 HĐĐT、泰國要 PromptPay。支付＋稅務合規缺一不可。
商業模式：不抽成。 中國廠商普遍主打「軟體年費＋硬體買斷、不抽成」，是重要賣點（如一桉約 1000 元／2 年＋硬體 160 元／桌）。

3中國市場

全球最成熟、最捲的市場，主軸是「無人值守自助共享球房」。

旗艦・武漢

球房雲掌櫃（德上雲）

42+ 功能模組，掃碼開檯、球桿櫃、人臉識別、AI 裁判自動判罰／精彩剪輯、點單大屏、多店統籌。宣稱 10+ 加盟品牌、5000+ 門店。

數位化

水滴智店（drip.im）

開檯計費、自助開檯、預約、客戶營銷＋智能硬體。

多場景

樂館+（ibuole）

SaaS 燈控計費，手機管理；台球/棋牌/自習室/羽球/籃球通用。

一體機

約個球啊（縱逑科技）

球控燈控一體機＋商家後台＋小程序，主攻公寓/社區/酒吧混合業態無人投放。

低價・不抽成

一桉科技

約 1000 元／2 年、硬體 160 元／桌、不抽成，案例多。

連鎖/低預算

商米・易捷・千佰・愛球客・威宏

連鎖大店或低預算方案常見品牌。

📈 趨勢：2025 年行動支付滲透率 >90%、無人經濟規模破 3000 億元，自助掃碼撞球桌加速取代傳統投幣式；硬體標配 4G 流量卡；CSDN 上甚至有全開源自助球房系統，降低山寨門檻。「助教經濟」（教練排班/抽成）是中國特有重點。

關鍵設計決策

多租戶

單庫＋tenant_id 行級隔離（Postgres RLS），MVP 成本最低；規模化後再分庫。

邊緣閘道 Edge

每店一台小主機，快取費率與會員餘額；斷網仍能開檯/結檯/離線收款，復網補同步。

設備走 MQTT

燈控盒↔Edge↔雲端，輕量即時、可離線佇列、QoS 保證。

在地化介接層

支付與電子發票全部抽象為 adapter，新國家只加 adapter，不動核心。

11IoT 燈控盒設計（核心硬體）

「開檯才亮燈」是亞洲防作弊的物理基礎，也是與純軟體競品的硬差異。

硬體規格

ESP32 主控（Wi-Fi+BLE）；6/8 路繼電器；Wi-Fi 為主、4G/RS-485 備援；實體按鈕人工接管；斷電狀態記憶；每盒唯一憑證＋TLS。

通訊拓撲

雲端 MQTT → Edge → 燈控盒，回 ACK。斷網時 Edge 直接控盒、交易記本地、復網補傳。

MQTT Topic（多租戶隔離）

t/{tenant_id}/store/{store_id}/dev/{device_id}/cmd     # 雲→設備 指令
t/{tenant_id}/store/{store_id}/dev/{device_id}/state   # 設備→雲 狀態
t/{tenant_id}/store/{store_id}/dev/{device_id}/hb      # 心跳 離線偵測

開檯指令（雲端 → 燈控盒）

{
  "msg_id": "c-9f3a...",          // 冪等鍵, 設備去重
  "action": "ON",                 // ON / OFF / PULSE
  "relay": 1,                     // 第幾路繼電器
  "session_id": "S-20260624-007", // 對應一筆開檯
  "expire_s": 14400,              // 逾時自動斷, 防忘關
  "sign": "HMAC-SHA256(payload, device_key)"
}

防作弊與安全機制

① 狀態以雲端 session 為準

  • 燈亮必對應有效 session；對不上 → 告警「未授權亮燈」

② 心跳離線偵測

  • 斷線即後台標紅，防「拔網私下營業」

③ 簽章＋冪等

  • HMAC 防偽造；msg_id 去重防重放

④ 逾時自動斷電

  • expire_s 安全上限，防忘記結檯漏計

⑤ 手動操作回報

  • 實體按鈕也回報雲端留痕（人工接管 ≠ 不記帳）

⑥ 每日對帳

  • 「亮燈時長 vs 計費時長」自動比對，差異即風險清單
